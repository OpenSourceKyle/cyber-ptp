```bash
### Network Discovery & Evasion (Nmap)

#### Basic Port Scans

#### TCP SYN Scan (Stealth Scan)
# Performs a half-open scan that is less likely to be logged.
nmap -sS -p <TARGET_PORT> <TARGET_IP>

#### TCP Connect Scan
# Completes the three-way handshake; more likely to be detected but more reliable.
nmap -sT -p <TARGET_PORT> <TARGET_IP>

#### UDP Scan
# Scans for open UDP ports; often much slower than TCP scans.
nmap -sU -p <TARGET_PORT> <TARGET_IP>

#### TCP ACK Scan
# Used to map out firewall rulesets by determining if they are stateful or not.
nmap -sA -p <TARGET_PORT> <TARGET_IP>

#### Service & OS Detection

#### Service & Version Detection
# Probes open ports to determine the service and version information.
nmap -sV <TARGET_IP>

#### OS Detection
# Attempts to determine the operating system of the target. 
# Specifying an open port can speed this up.
nmap -O -p <OPEN_PORT> <TARGET_IP>

#### Aggressive Scan
# A noisy scan that enables OS detection (`-O`), version detection (`-sV`), 
# default script scanning (`-sC`), and a traceroute.
nmap -A <TARGET_IP>

#### Firewall/IDS Evasion & Stealth Scans

#### Idle (Zombie) Scan (Automated)
# Performs a completely blind port scan by using Nmap's automated engine 
# to probe a zombie host.
nmap -sI <ZOMBIE_IP>:<ZOMBIE_PORT> -p <TARGET_PORT> <TARGET_IP> -Pn

#### Hybrid Idle Scan (Manual Verification)
# Use hping3 to monitor the zombie's IPID in one terminal, while Nmap sends 
# the spoofed packets from another. This is useful for manual verification 
# when automated scans may fail.
#
# Terminal 1: Monitor the Zombie's IPID with hping3
hping3 -r -S -p <OPEN_PORT> <ZOMBIE_IP>
#
# Terminal 2: Send spoofed packets with Nmap
nmap -sS -Pn -n -e <INTERFACE> -S <ZOMBIE_IP> <TARGET_IP> -p <TARGET_PORT>

#### Packet Fragmentation
# Splits packets into smaller fragments, which can evade less 
# sophisticated packet-inspection firewalls.
nmap -f --data-length <LENGTH> -sS -p <TARGET_PORT> <TARGET_IP> -Pn

#### Decoy Scan
# Hides the attacker's real IP address by launching the scan from 
# multiple spoofed decoy IPs.
nmap -sS -D <DECOY_IP_1>,<DECOY_IP_2>,ME <TARGET_IP>

#### Randomized Decoy Scan
# Uses a specified number of random, non-existent IP addresses as decoys.
nmap -sS -D RND:<NUMBER> <TARGET_IP>

#### Timing Template
# Slows down the scan to evade IDS/IPS based on scan rate detection. 
# T0 is the slowest, T5 is the fastest.
nmap -sS -T<0-5> <TARGET_IP>

#### Source Port Specification
# Specifies a specific source port (e.g., 53 for DNS) to bypass simple firewall rules.
nmap -sS --source-port <SOURCE_PORT> <TARGET_IP>

#### MAC Address Spoofing
# Changes the source MAC address to a vendor name (`apple`), 
# a random address (`0`), or a specific address.
nmap --spoof-mac <MAC_ADDR|VENDOR|0> -p <TARGET_PORT> <TARGET_IP> -Pn

#### Nmap Scripting Engine (NSE)
# These commands are kept separate as requested.

#### Identify Idle Scan "Zombie"
# Uses the `ipidseq` script to test if a host has an incremental IP ID, 
# making it a good zombie candidate.
nmap --script ipidseq -p <OPEN_PORT> <ZOMBIE_IP>

#### Whois Domain Query
# Retrieves `whois` information for the target domain.
nmap -sn --script whois-domain <TARGET_DOMAIN>

#### SMB OS Discovery
# Discovers the operating system, computer name, and domain of a target 
# via the SMB protocol.
nmap -p 445 --script smb-os-discovery <TARGET_IP>

#### SMB Share Enumeration
# Attempts to list available SMB shares on the target.
nmap -p 445 --script smb-enum-shares <TARGET_IP>

---
### Banner Grabbing & Passive Fingerprinting

#### Banner Grabbing
# Connects to a specific port to retrieve the service banner, 
# which can reveal version and type information.
# Using ncat
ncat <TARGET_IP> <TARGET_PORT>
# Using netcat
nc <TARGET_IP> <TARGET_PORT>
# Using telnet
telnet <TARGET_IP> <TARGET_PORT>

#### Passive OS Fingerprinting (p0f)
# Analyzes network traffic passively without sending any packets to the target, 
# allowing for stealthy OS identification.
./p0f -i <INTERFACE>
```
